# SoloChef 用户体验地图

> 版本：v1.0 | 更新：2026-06-03

---

## 一、用户画像

### Persona A：独居烹饪新手
- 25-35 岁，独居，刚学做饭
- 痛点：不知道每天吃什么、做菜手忙脚乱、忘了买什么食材
- 目标：有计划的、不焦虑的做饭体验
- 使用频率：每周 3-5 次（晚餐为主）

### Persona B：家庭主厨/煮夫
- 30-45 岁，负责家庭日常三餐
- 痛点：重复做同样几道菜、一周买一次菜但经常忘了有什么、需要批量规划
- 目标：高效管理家庭厨房，探索新菜式
- 使用频率：每天 1-2 次

### Persona C：厨房爱好者
- 25-40 岁，享受烹饪过程本身
- 痛点：想记录自己的食谱、想尝试更多复杂菜式
- 目标：建立个人菜谱库，追踪烹饪数据
- 使用频率：每周 3-7 次

## 二、核心用户旅程（Persona A 视角）

### 阶段 0 — 发现与安装
```
场景：小红书上刷到"独厨"推荐帖
情绪：好奇 → 期待
动作：下载/打开应用
触点：应用商店/网页入口
```

### 阶段 1 — 首次使用（空状态）

```
  情绪
   ↑ 期待
   │  ┌─ 打开应用，看到干净的首页
  5│  │   空状态展示："独厨SoloChef / 你的精品线上厨房"
   │  │   ┌─ 没有菜谱，只有"新建菜谱"按钮
  4│  │   │  ┌─ 点进去，开始创建第一道菜谱
   │  │   │  │
  3│  │   │  │  发现需要填封面、食材、步骤…好多
   │  │   │  │  ┌─ 耐心填完 3 个食材 + 4 个步骤
  2│  │   │  │  │  ┌─ 点"保存发布菜谱"
   │  │   │  │  │  │
  1│  │   │  │  │  │  回到首页，看到菜谱卡片出现了！
   │──┼──┼──┼──┼──┼──┼──┼──→ 时间
   └──┴──┴──┴──┴──┴──┴──┘  触点
        首次创建菜谱
        输入成本较高
        缺少引导/Mock数据
```

**情绪趋势**：😊→😐→😤→😊
**痛点**：
- P1: 空状态缺乏引导，不知道从哪开始
- P2: 首次创建菜谱输入成本高（约需 3-5 分钟）
- P3: 没有示例菜谱可供参考

**机会**：
- 提供 3-5 道预置菜谱（如"西红柿炒鸡蛋""煎牛排"等高频菜）
- 首次进入时引导式创建（分步向导）
- AI 辅助生成菜谱（利用已有 Gemini API 依赖）

### 阶段 2 — 规划今日菜单

```
  情绪
   ↑
  5│         ┌─ 打开首页
   │         │  "今晚吃什么？" 看到随机选菜按钮
  4│         │  ┌─ 点击随机，出来一道菜
   │         │  │  ┌─ 不满意，再点，又出来一道
  3│         │  │  │  ┌─ 这道不错！点"下单"
   │         │  │  │  │
  2│         │  │  │  │  首页顶部变化："物料筹备中"
   │         │  │  │  │
  1│         │  │  │  │  ← 焦虑感消失了
   │─────────┼──┼──┼──┼──┼──→ 时间
   首页浏览  随机    下单  进入
             选菜    确认  批次
```

**情绪趋势**：😐→😀→😃
**触点**：Dashboard（推荐网格 + 随机选菜 + 工艺过滤）
**亮点**：
- ✅ 随机选菜降低决策成本
- ✅ 推荐算法（主推菜 + 一周未做）提供理性建议
- ✅ 工艺标签过滤帮助快速缩小范围

**痛点**：
- P2: 多菜谱点单需要进入 OrderEngine，路径偏长
- P2: 随机选菜无法排除用户明确不想要的菜

**机会**：
- 首页直接显示"快速下单"（单菜一键跳转）
- 随机选菜增加"换一个"按钮和排除逻辑

---

### 阶段 3 — 采购物料（Picking）

```
  情绪
   ↑
  5│                   ┌─ 到达批次详情页
   │                   │  "批量采购清单 (Batch BOM)"
  4│                   │  看到肉/菜/调 三分区
   │                   │  ┌─ 逐项勾选 √
  3│                   │  │  √ 猪里脊 300g
   │                   │  │  √ 青椒 2个
  2│                   │  │  √ 生抽 1勺
   │                   │  │  ┌─ 全部完成！
  1│                   │  └─┘  │  点"采购完成"
   │                   │       │  进度条从 1/3 到 2/3
   │───────────────────┼───────┼──→ 时间
   进入批次             逐项    确认
                       勾选    采购
```

**情绪趋势**：😊→😊→😃
**触点**：BatchDetail (Picking mode)
**亮点**：
- ✅ BOM 自动聚合去重——两个菜都用了"盐"，自动合并
- ✅ 三大分类清晰，去超市按路线采购
- ✅ 勾选反馈（变灰、划掉、打勾动画）

**痛点**：
- P2: 目前不支持"缺料替换"建议
- P3: 食材图片依赖用户手动上传，空状态较多

**机会**：
- AI 驱动的"缺料替代方案"推荐
- 关联电商平台一键加购（如叮咚买菜、盒马 API）

---

### 阶段 4 — 烹饪执行（Execution）

```
  情绪
   ↑          ┌─ 做好准备工作，点击"开始加工"
  5│          │  进入烹饪计时界面
   │          │  ┌─ Step 1: 腌制肉片 (10分钟)
  4│          │  │  倒计时开始 → 10:00 ... 5:00 ... 0:03
   │          │  │  ┌─ 手机震动 + 屏幕脉冲 → "叮！"
  3│          │  │  │  自动跳下一步
   │          │  │  │  ┌─ Step 2: 炒青椒 (5分钟)
  2│          │  │  │  │  ┌─ 太忙了！按 VOL+ 暂停
   │          │  │  │  │  │  处理完继续
  1│          │  │  │  │  │  ┌─ 最后一步完成！
   │──────────┼──┼──┼──┼──┼──┼──→ 时间
   进入执行  腌制   到  炒   暂停 全部
                    期      恢复 完成
```

**情绪趋势**：😃→😤(分心)→😌(暂停)→😃
**触点**：ExecutionView
**亮点**：
- ✅ 大字号倒计时（移动端友好，离开 2 米也能看清）
- ✅ 物理按键映射（VOL+/VOL- 控制暂停/跳过，湿手友好）
- ✅ WakeLock 防止锁屏
- ✅ 等待步骤半透明展示，视觉上区分优先级
- ✅ 步骤图解（图片 + 全屏查看）

**痛点**：
- P1: Android 端物理按键映射未实现
- P2: 音频提示依赖网络（Mixkit CDN）
- P2: 懒人模式仅显示文字建议，无实质功能

**机会**：
- 离线音频包（本地 .mp3 或系统通知音）
- 语音播报步骤（"下一个步骤：炒青椒，5分钟"）
- 多设备协同（手机计时 + 平板看步骤）

---

### 阶段 5 — 完成反馈（Feedback）

```
  情绪
   ↑
  5│                      ┌─ 烹饪结束，自动跳转
   │                      │  一张热敏纸小票慢慢展开
  4│                      │  "solochef / 独厨（蓝星旗舰店）"
   │                      │  ┌─ 取票号 #0042（递增！）
  3│                      │  │  名字、价格、税费 0.00
   │                      │  │  ┌─ "关火，出锅。趁热尝第一口…"
  2│                      │  │  │  随机温暖寄语出现
   │                      │  │  │  ┌─ 点"打印小票"
  1│                      │  │  │  │  回到首页
   │──────────────────────┼──┼──┼──┼──┼──→ 时间
   自动跳转              小票  寄语  返回
                         展开       首页
```

**情绪趋势**：😃→😍→😊
**触点**：FeedbackView
**亮点**：
- ✅ 游戏化 "取票号" 设计（递增 counter 增强成就感）
- ✅ 模拟热敏纸票据（锯齿边、斑驳纹理、手写体）
- ✅ 随机温暖寄语（6 句备选，每次不同）
- ✅ Android 端可"保存小票"为图片到相册

**痛点**：
- P2: 热敏纸尺寸固定，长菜名截断
- P3: 缺少社交分享功能（朋友圈/小红书分享卡片）

**机会**：
- 一键生成分享卡片（菜谱图 + 取票号 + 打卡天数）
- 连续 7 天打卡特殊小票样式
- 成就系统徽章（如"一周全勤""首次尝试新菜系"）

## 三、多菜谱批量烹饪旅程

```
情绪
 ↑
5│  OrderEngine → 选 3 道菜 → 生成计划
 │  ┌─ Picking: 聚合 BOM（猪肉300g + 猪肉200g = 500g）
4│  │  ┌─ Processing: 3 张菜谱卡片
 │  │  │  ┌─ 先做 A → 完成 → Feedback
3│  │  │  │  ┌─ 回到 Processing → 再做 B → Feedback
 │  │  │  │  │  ┌─ 回到 Processing → 再做 C → Feedback
2│  │  │  │  │  │  ┌─ 一键归档 → 批量热敏纸
 │  │  │  │  │  │  │  小票显示：3道菜 × 份数 × 价格
1│  │  │  │  │  │  │
 │──┼──┼──┼──┼──┼──┼──┼──→ 时间
   聚合BOM  A→    B→    C→   归档
       完成 完成   完成   完成
```

**情绪趋势**：😊→😊→😤(重复)→😊→😃
**亮点**：
- ✅ BOM 智能聚合（去重合并）
- ✅ "一键归档所有"快捷键
- ✅ 批量热敏纸聚合计算

**痛点**：
- P2: 每完成一道菜都要走 Feedback→Processing 来回，重复感强
- P1: 中间关闭应用再打开，批次可能丢失（依赖存储可靠性）

**机会**：
- Processing 页面直接内联 "完成" 弹窗（不走全屏 Feedback）
- 批量模式最后一次性展示完整收据

## 六、关键时刻（Moments of Truth）

1. **首次创建菜谱** — 决定了用户是否愿意继续使用（当前缺乏引导，有流失风险）
2. **第一次计时完成** — "叮！" + 自动跳转的爽感，是形成习惯的关键
3. **取票号 +1** — 游戏化的核心锚点，每次看到数字增长都有正向反馈
4. **连续 7 天** — 解锁"烟火气守护者"称号，社交分享欲望最强的时间点
5. **数据同步失败** — 菜谱丢失是最严重的信任危机，必须确保数据可靠性

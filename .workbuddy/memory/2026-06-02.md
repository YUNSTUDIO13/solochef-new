## 量输入框文本裁剪 + PickingScreen底部 + Analytics强制重载 (15:43, BUILD SUCCESSFUL)

### 物料"量"输入框文本被裁剪 (CreateRecipeScreen.kt)
- **根因**: `OutlinedTextField` 自带内部 padding(~32dp)，`width(56.dp)` 下可用文本区仅 24dp，中文"适量"（14sp×2字≈28dp）被截断
- **修复**: 量、单位输入框由 `OutlinedTextField` 改为 `BasicTextField` + `decorationBox = Center`，消除内边距，文本完整显示
  - 量：`BasicTextField` width=52dp, fontSize=14sp, divider 分隔符
  - 单位：`BasicTextField` width=40dp, fontSize=12sp

### PickingScreen 底部底色 + 滚动区域 (PickingScreen.kt)
- **底部全宽底色**: footer `Surface` 移除 `padding(32.dp)`，改为 `fillMaxWidth()` 全宽 + `shadowElevation=8.dp` 阴影
- **内容滚动**: scroll Column 添加 `padding(bottom = 120.dp)` 确保底部物料可滚动至按钮上方，不被遮挡

### Analytics 数据大盘仅首次有效 (AnalyticsScreen.kt)
- **根因定位**: Navigation Compose 可能复用 NavBackStackEntry，`LaunchedEffect(Unit)` 仅在首次 composition 触发；后续导航回 Analytics 时 ViewModel 未被重建，数据不刷新
- **修复**: `LaunchedEffect(Unit) { viewModel.reload() }` → `LaunchedEffect(lifecycleOwner) { lifecycle.repeatOnLifecycle(STARTED) { viewModel.reload() } }`
  - 每次屏幕进入 STARTED 状态（从不可见变为可见）时强制从存储重新加载
  - 确保每次进入 Analytics 页面都获取最新 last_cooked_at 数据

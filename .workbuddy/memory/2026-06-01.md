## Phase 6 — Dock Bar + 标签点击 + 布局对齐修复 (10:57, BUILD SUCCESSFUL)

用户截图对比 Web vs Android，指出三处问题：

### Dock Bar 高度/位置 (MainActivity.kt)
- **根因**: Web FAB `bottom-24`(96px) 与 Nav `bottom-6`(24px) 间距 24px，Android Scaffold `FabPosition.Center` 默认仅 16dp
- **修复**: FAB Surface 加 `Modifier.padding(bottom = 8.dp)`，将间距从 16dp 扩至 24dp

### CreateRecipeScreen 标签不响应点击
- **根因**: `tags.value -= t` 直接 mutate `MutableSet`，引用未变 → `mutableStateOf` 不触发 recomposition
- **修复**: 2处工艺标签 + 1处菜系标签 + 1处自定义标签，全部改为 `tags.value = (tags.value - t).toMutableSet()` / `tags.value = (tags.value + t).toMutableSet()`

### 标签布局错位 ("二次存在错位")
- **根因**: 标签用硬编码 4 列 grid (`rowItems.size == 4`)，Web 用 `flex flex-wrap gap-2` 自然换行
- **修复**: 工艺(8)和菜系(21)标签段落均由 `Column { Row { ... } }` 改为 `FlowRow`（`@OptIn(ExperimentalLayoutApi::class)`），匹配 Web 自然换行行为
